## 🚀 Как использовать

### Первый запуск:

1. **Установите зависимости** (если еще не установлены):
   ```bash
   pip install -r requirements.txt
   ```

2. **Соберите приложение**:
   ```bash
   BUILD.bat
   ```
   Это создаст:
   - Иконку (если её нет)
   - `dist/Telegraf.exe` с иконкой

3. **Создайте установщик** (опционально):
   ```bash
   BUILD_INSTALLER.bat
   ```
   Требуется: Inno Setup 6
   Создаст: `installer/Telegraf_Setup.exe`

4. **Запустите**:
   ```bash
   RUN.bat
   ```
   Или запустите `dist/Telegraf.exe`

5. **Зарегистрируйтесь**:
   - Нажмите "Регистрация"
   - Введите имя пользователя (мин. 3 символа)
   - Введите пароль (мин. 4 символа)
   - Нажмите "Зарегистрироваться"

6. **Войдите**:
   - Введите имя пользователя
   - Введите пароль
   - Нажмите "Войти"

## 📤 Как поделиться с друзьями

### Вариант 1: Отправить .exe
```bash
BUILD.bat
```
Отправьте друзьям файл `dist/Telegraf.exe` (~50 МБ)

### Вариант 2: Создать установщик
```bash
BUILD_INSTALLER.bat
```
Отправьте друзьям `installer/Telegraf_Setup.exe` (~50-100 МБ)

### Вариант 3: Развернуть сервер в интернете

**Для общения через интернет нужен публичный сервер:**

1. **Зарегистрируйтесь на Render.com** (бесплатно)
   - https://render.com

2. **Создайте Web Service**
   - Загрузите папку `relay`
   - Runtime: Python 3
   - Build: `pip install -r requirements.txt`
   - Start: `python server.py`

3. **Получите URL**
   - Например: `https://telegraf-relay.onrender.com`
   - Замените на: `wss://telegraf-relay.onrender.com`

4. **Обновите config.json**
   ```json
   {
       "relay_server": "wss://telegraf-relay.onrender.com"
   }
   ```

5. **Пересоберите**
   ```bash
   BUILD.bat
   ```

6. **Поделитесь с друзьями**
   - Отправьте `Telegraf.exe` или `Telegraf_Setup.exe`
   - Все будут подключаться к вашему серверу

**Подробная инструкция**: `КАК_ЗАПУСТИТЬ.md`

## 🎯 Основные функции

### Регистрация:
- Обязательная для всех пользователей
- Имя пользователя: минимум 3 символа, уникальное
- Пароль: минимум 4 символа, хешируется SHA-256
- База данных: SQLite в `data/users.db`

### Вход:
- Проверка имени пользователя и пароля
- Сообщения об ошибках на русском
- Автоматический вход после регистрации

### Иконка:
- Современный дизайн с градиентом
- Автоматически создается при сборке
- Используется в .exe и установщике

### Установщик:
- Выбор папки установки
- Ярлыки на рабочем столе и в меню Пуск
- Опция сохранить/удалить данные при удалении
- Русский язык

## 🔒 Безопасность

### Что защищено:
- ✅ Пароли хешируются (SHA-256)
- ✅ Сообщения шифруются end-to-end (AES-256)
- ✅ Публичные ключи обмениваются автоматически
- ✅ Сервер не может прочитать сообщения
- ✅ База данных локальная (SQLite)

### Рекомендации:
- Используйте сложные пароли
- Не делитесь паролями
- Регулярно делайте бэкап `data/users.db`
- Используйте свой сервер для приватности

## 🛠️ Решение проблем

### "Пользователь уже существует"
→ Выберите другое имя пользователя

### "Неверное имя пользователя или пароль"
→ Проверьте правильность ввода
→ Убедитесь, что вы зарегистрированы

### "Забыл пароль"
→ Удалите `data/users.db` и зарегистрируйтесь заново

### "Иконка не отображается"
→ Запустите `python scripts/create_icon.py`
→ Пересоберите с `BUILD.bat`

### "Ошибка при сборке"
→ Установите зависимости: `pip install -r requirements.txt`
→ Проверьте Python версию: `python --version` (должен быть 3.11+)
