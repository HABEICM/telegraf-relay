## 🏠 Быстрый старт (локальная сеть)

### Если вы в одной Wi-Fi сети с друзьями:

**Шаг 1: Запустите сервер (один человек)**
```bash
START_RELAY.bat
```
Сервер запустится на `ws://localhost:8765`

**Шаг 2: Узнайте свой IP адрес**
```bash
ipconfig
```
Найдите строку "IPv4 Address", например: `192.168.1.100`

**Шаг 3: Друзья должны изменить config.json**
Файл: `config/config.json`
```json
{
    "relay_server": "ws://192.168.1.100:8765"
}
```
Замените `192.168.1.100` на IP адрес человека, который запустил сервер.

**Шаг 4: Все запускают клиент**
```bash
RUN.bat
```

**Шаг 5: Регистрация**
- При первом запуске нажмите "Регистрация"
- Введите имя пользователя (минимум 3 символа)
- Введите пароль (минимум 4 символа)
- Нажмите "Зарегистрироваться"

**Шаг 6: Общайтесь!**
- Выберите пользователя из списка слева
- Пишите сообщения
- Все сообщения зашифрованы end-to-end

## 🌍 Запуск через интернет

### Для общения с друзьями из других городов нужен публичный сервер.

### Вариант 1: Render.com (Бесплатно, рекомендуется)

**Шаг 1: Создайте аккаунт на Render.com**
- Перейдите на https://render.com
- Зарегистрируйтесь (можно через GitHub)

**Шаг 2: Создайте новый Web Service**
1. Нажмите "New +" → "Web Service"
2. Подключите GitHub репозиторий или загрузите папку `relay`
3. Настройки:
   - **Name**: `telegraf-relay` (или любое имя)
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python server.py`
   - **Plan**: Free

**Шаг 3: Дождитесь деплоя**
- Render автоматически установит зависимости и запустит сервер
- Это займет 2-5 минут

**Шаг 4: Скопируйте URL**
- После деплоя вы получите URL типа: `https://telegraf-relay.onrender.com`
- Замените `https://` на `wss://`
- Итоговый URL: `wss://telegraf-relay.onrender.com`

**Шаг 5: Обновите config.json**
```json
{
    "relay_server": "wss://telegraf-relay.onrender.com",
    "fallback_servers": []
}
```

**Шаг 6: Пересоберите приложение**
```bash
BUILD.bat
```

**Шаг 7: Поделитесь с друзьями**
- Отправьте друзьям файл `dist/Telegraf.exe`
- Или создайте установщик (см. ниже)

---

### Вариант 2: Railway.app (Альтернатива)

**Шаг 1: Создайте аккаунт на Railway.app**
- Перейдите на https://railway.app
- Зарегистрируйтесь через GitHub

**Шаг 2: Создайте новый проект**
1. Нажмите "New Project"
2. Выберите "Deploy from GitHub repo"
3. Выберите репозиторий с папкой `relay`

**Шаг 3: Настройте переменные окружения**
- Добавьте переменную `PORT` = `8765`

**Шаг 4: Получите URL**
- Railway автоматически создаст публичный URL
- Используйте его в формате `wss://your-app.railway.app`

---

### Вариант 3: Ngrok (Для тестирования)

**Быстрый способ сделать локальный сервер доступным в интернете:**

**Шаг 1: Скачайте ngrok**
- https://ngrok.com/download
- Зарегистрируйтесь и получите токен

**Шаг 2: Запустите локальный сервер**
```bash
START_RELAY.bat
```

**Шаг 3: Запустите ngrok**
```bash
ngrok tcp 8765
```

**Шаг 4: Скопируйте URL**
- Ngrok покажет URL типа: `tcp://0.tcp.ngrok.io:12345`
- Используйте его в config.json: `ws://0.tcp.ngrok.io:12345`

⚠️ **Внимание**: Бесплатный ngrok отключается через 2 часа, нужно перезапускать.

## 📦 Создание установщика

### Для удобной установки на компьютеры друзей:

**Требования:**
- Inno Setup 6 (скачать: https://jrsoftware.org/isdl.php)

**Шаг 1: Установите Inno Setup**

**Шаг 2: Соберите приложение**
```bash
BUILD.bat
```

**Шаг 3: Создайте установщик**
```bash
BUILD_INSTALLER.bat
```

**Шаг 4: Найдите установщик**
- Файл: `installer/Telegraf_Setup.exe`
- Размер: ~50-100 МБ

**Шаг 5: Поделитесь с друзьями**
- Отправьте `Telegraf_Setup.exe` через:
  - Google Drive
  - Dropbox
  - WeTransfer
  - Telegram/WhatsApp (если размер позволяет)

**Установка у друзей:**
1. Запустить `Telegraf_Setup.exe`
2. Выбрать папку установки (можно изменить)
3. Выбрать опции:
   - ✅ Создать ярлык на рабочем столе
   - ✅ Создать ярлык в меню Пуск
4. Нажать "Установить"
5. Запустить Telegraf
6. Зарегистрироваться

## 🔐 Регистрация и вход

### Первый запуск - Регистрация

**Шаг 1: Откройте Telegraf**
- Появится окно входа

**Шаг 2: Нажмите "Регистрация"**

**Шаг 3: Заполните данные**
- **Имя пользователя**: минимум 3 символа (например: `alex`, `maria123`)
- **Пароль**: минимум 4 символа (например: `pass1234`)

**Шаг 4: Нажмите "Зарегистрироваться"**
- Если имя занято, выберите другое
- После успешной регистрации вы автоматически войдете

### Повторный вход

**Шаг 1: Откройте Telegraf**

**Шаг 2: Введите данные**
- Имя пользователя
- Пароль

**Шаг 3: Нажмите "Войти"**

### Важно:
- ✅ Каждый пользователь должен зарегистрироваться
- ✅ Данные хранятся локально в `data/users.db`
- ✅ Пароли хешируются (SHA-256)
- ✅ Без регистрации войти нельзя
- ⚠️ Если забыли пароль - нужно удалить `data/users.db` и зарегистрироваться заново

## 🛠️ Решение проблем

### Проблема: "Не могу подключиться к серверу"

**Решение:**
1. Проверьте, что сервер запущен (`START_RELAY.bat`)
2. Проверьте `config/config.json` - правильный ли URL
3. Проверьте брандмауэр Windows:
   - Панель управления → Брандмауэр Windows
   - Разрешить приложение → Добавить Python
4. Если используете Render.com - подождите 1-2 минуты после деплоя

### Проблема: "Пользователь уже существует"

**Решение:**
- Выберите другое имя пользователя
- Или удалите `data/users.db` и зарегистрируйтесь заново

### Проблема: "Неверное имя пользователя или пароль"

**Решение:**
- Проверьте правильность ввода
- Убедитесь, что вы зарегистрированы
- Если забыли пароль - удалите `data/users.db`

### Проблема: "Не вижу других пользователей"

**Решение:**
1. Убедитесь, что все подключены к одному серверу
2. Проверьте, что у всех одинаковый `relay_server` в config.json
3. Перезапустите приложение

### Проблема: "Сообщения не доходят"

**Решение:**
1. Проверьте интернет-соединение
2. Убедитесь, что получатель онлайн
3. Проверьте логи в папке `logs/`

### Проблема: "Окно не открывается"

**Решение:**
1. Проверьте логи в `logs/telegraf_*.log`
2. Убедитесь, что установлены все зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Попробуйте запустить через `RUN.bat` вместо `Telegraf.exe`

### Проблема: "Ошибка при сборке"

**Решение:**
1. Обновите PyInstaller:
   ```bash
   pip install --upgrade pyinstaller
   ```
2. Проверьте, что все файлы на месте
3. Запустите `BUILD.bat` от имени администратора

## 🔒 Безопасность

### Что защищено:
- ✅ Сообщения шифруются end-to-end (AES-256)
- ✅ Пароли хешируются (SHA-256)
- ✅ Публичные ключи обмениваются автоматически
- ✅ Сервер не может прочитать сообщения

### Что НЕ защищено:
- ⚠️ Метаданные (кто с кем общается, когда)
- ⚠️ Имена пользователей видны всем
- ⚠️ Статус онлайн/оффлайн виден всем

### Рекомендации:
- Используйте сложные пароли
- Не делитесь паролями
- Используйте свой сервер для приватности
- Регулярно обновляйте приложение
